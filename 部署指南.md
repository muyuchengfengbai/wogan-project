# 沃柑平台本地部署完整指南

## 📋 系统要求

- Node.js >= 18.x (你已安装 v18.19.1 ✅)
- PostgreSQL >= 13.x
- Bun (推荐) 或 npm

---

## 🚀 快速部署步骤

### 步骤 1: 安装 Bun (可选但推荐)

```bash
# 安装 Bun - 比 npm 快 10 倍
curl -fsSL https://bun.sh/install | bash

# 重新加载 shell 配置
source ~/.bashrc  # 或 source ~/.zshrc
```

### 步骤 2: 安装项目依赖

```bash
cd /home/xiaobai/code/biye/woganproject/anything/apps/web

# 使用 Bun (推荐)
bun install

# 或使用 npm
npm install
```

### 步骤 3: 配置数据库

#### 方案 A: 使用 Neon 云数据库 (最简单，推荐用于演示)

1. 访问 https://neon.tech 注册免费账号
2. 创建新项目 (Project Name: wogan-platform)
3. 复制连接字符串 (类似: `postgresql://user:pass@ep-xxx.neon.tech/dbname`)
4. 编辑 `.env` 文件:

```bash
nano /home/xiaobai/code/biye/woganproject/anything/apps/web/.env
```

添加或修改这一行:
```env
DATABASE_URL=postgresql://你的连接字符串
```

#### 方案 B: 本地安装 PostgreSQL

```bash
# 1. 安装 PostgreSQL
sudo apt update
sudo apt install postgresql postgresql-contrib -y

# 2. 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 3. 创建数据库和用户
sudo -u postgres psql << EOF
CREATE DATABASE wogan_db;
CREATE USER wogan_user WITH PASSWORD 'wogan123456';
GRANT ALL PRIVILEGES ON DATABASE wogan_db TO wogan_user;
\q
EOF

# 4. 配置 .env 文件
echo "DATABASE_URL=postgresql://wogan_user:wogan123456@localhost:5432/wogan_db" > .env
```

### 步骤 4: 初始化数据库表结构

```bash
# 进入项目目录
cd /home/xiaobai/code/biye/woganproject/anything/apps/web

# 执行初始化脚本 (已为你生成)
psql $DATABASE_URL -f init_database.sql

# 或者如果使用本地数据库
psql -U wogan_user -d wogan_db -f init_database.sql
```

这将创建以下表并插入示例数据:
- ✅ products (商品表) - 3条示例数据
- ✅ orders (订单表) - 3条示例数据
- ✅ order_items (订单明细表)
- ✅ traceability (溯源记录表) - 5条示例数据
- ✅ orchard_sensors (传感器数据表) - 3条示例数据

### 步骤 5: 启动开发服务器

```bash
# 使用 Bun
bun run dev

# 或使用 npm
npm run dev
```

服务器将在 **http://localhost:4000** 启动

## 🛠️ 常见问题排查

### 问题 1: 数据库连接失败

**错误信息**: `No database connection string was provided`

**解决方案**:
```bash
# 检查 .env 文件是否存在
cat /home/xiaobai/code/biye/woganproject/anything/apps/web/.env

# 确保包含 DATABASE_URL
# 如果没有，手动创建
echo "DATABASE_URL=你的数据库连接字符串" > .env
```

### 问题 2: 端口 4000 被占用

**错误信息**: `Port 4000 is already in use`

**解决方案**:
```bash
# 查找占用端口的进程
sudo lsof -i :4000

# 杀死进程
sudo kill -9 <PID>

# 或修改端口 (编辑 vite.config.ts)
```

### 问题 3: 依赖安装失败

**解决方案**:
```bash
# 清理缓存重新安装
rm -rf node_modules bun.lock
bun install

# 或使用 npm
rm -rf node_modules package-lock.json
npm install
```

### 问题 4: 数据库表不存在

**错误信息**: `relation "products" does not exist`

**解决方案**:
```bash
# 重新执行初始化脚本
psql $DATABASE_URL -f init_database.sql
```
