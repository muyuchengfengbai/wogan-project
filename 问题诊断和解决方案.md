### 问题 2：收货地址在结算页面使用

**代码位置**：
- 地址管理：`src/app/profile/addresses/page.jsx`
- 结算页面：`src/app/checkout/page.jsx` 第 31-56 行

**实现逻辑**：
1. 在地址管理页面保存地址到 `localStorage.user_addresses`
2. 结算页面自动读取 `localStorage.user_addresses`
3. 自动选择默认地址或第一个地址
4. 用户可以选择其他地址或手动输入

**测试步骤**：
1. 访问 http://localhost:4000/profile/addresses
2. 点击"添加新地址"

## 常见问题排查

### 1. 浏览器缓存问题

**症状**：代码已更新，但功能还是不工作

**解决方法**：
```
按 Ctrl+Shift+R（Windows）或 Cmd+Shift+R（Mac）强制刷新
或者清除浏览器缓存
或者在无痕模式下测试
```

### 2. localStorage 被禁用

**症状**：所有功能都不工作

**检查方法**：
```javascript
// 在浏览器控制台输入
typeof localStorage
// 应该返回 "object"
// 如果返回 "undefined"，说明被禁用
```

**解决方法**：
- 检查浏览器设置，启用 Cookie 和网站数据
- 在无痕模式下测试
- 尝试其他浏览器

### 3. JavaScript 错误

**症状**：点击按钮没有反应

**检查方法**：
```
1. 按 F12 打开开发者工具
2. 切换到 Console 标签
3. 查看是否有红色错误信息
4. 截图错误信息
```

### 4. 浏览器扩展干扰

**症状**：某些功能不工作

**解决方法**：
- 禁用所有浏览器扩展
- 在无痕模式下测试

### 5. 点击位置错误

**症状**：点击收藏按钮没反应

**原因**：可能点击了商品图片而不是爱心图标

**解决方法**：
- 确保点击商品卡片**右上角**的爱心图标
- 爱心图标是一个独立的按钮，不是图片的一部分
