## 🚀 完整部署流程（6 大步骤）

### 步骤 1：配置开发环境 ⏱️ 2分钟

```bash
# 1. 切换到项目目录
cd /home/xiaobai/code/biye/woganproject/anything/apps/web

# 2. 配置 npm 使用国内镜像
npm config set registry https://registry.npmmirror.com

# 3. 验证配置
npm config get registry
# 应该显示：https://registry.npmmirror.com
```

---

### 步骤 2：准备部署文件 ⏱️ 1分钟

```bash
# 在项目目录执行
cd /home/xiaobai/code/biye/woganproject/anything/apps/web

# 一键创建所有配置文件
cat > create-config.sh << 'EOF'
#!/bin/bash

# 创建 vercel.json
cat > vercel.json << 'VERCEL'
{
  "buildCommand": "npm run build",
  "outputDirectory": "build/client",
  "devCommand": "npm run dev",
  "installCommand": "npm install"
}
VERCEL

# 创建 .vercelignore
cat > .vercelignore << 'IGNORE'
node_modules
.env
.env.local
*.log
pgdata
test-results
IGNORE

# 创建 .env.example
cat > .env.example << 'ENVEX'
DATABASE_URL=postgresql://user:password@host:5432/database
NODE_ENV=production
ENVEX

echo "✅ 配置文件创建完成"
EOF

chmod +x create-config.sh
bash create-config.sh
```

### 步骤 3：推送到 GitHub ⏱️ 5分钟

#### 3.1 初始化 Git

```bash
cd /home/xiaobai/code/biye/woganproject/anything/apps/web

# 初始化 Git
git init

# 添加所有文件
git add .

# 提交
git commit -m "Deploy to Vercel"
```

#### 3.2 创建 GitHub 仓库

1. **打开浏览器**：https://github.com/new

2. **填写信息**：
   - Repository name: `wogan-project`
   - 选择：`Public`
   - **不要勾选任何选项**

3. **点击**：`Create repository`

#### 3.3 推送代码

**替换 `YOUR_USERNAME` 为你的 GitHub 用户名**：

```bash
# 关联远程仓库
git remote add origin https://github.com/YOUR_USERNAME/wogan-project.git

# 重命名分支为 main
git branch -M main

# 推送代码
git push -u origin main
```

**如果提示需要登录**：
- 输入 GitHub 用户名
- 密码使用 Personal Access Token（不是 GitHub 密码）
- 获取 Token：https://github.com/settings/tokens → Generate new token (classic) → 勾选 `repo` → 生成并复制

### 步骤 4：创建 Neon 数据库 ⏱️ 5分钟

#### 4.1 注册并创建项目

1. **访问**：https://neon.tech
2. **点击**：`Sign Up` → `Continue with GitHub`
3. **授权** Neon
4. **点击**：`Create a project`
5. **填写**：
   - Project name: `wogan-db`
   - Region: `Asia Pacific (Singapore)`
6. **点击**：`Create project`

#### 4.2 获取连接字符串

1. 在项目页面找到 **Connection string**
2. 复制连接字符串（类似）：
   ```
   postgresql://username:password@ep-xxx.region.aws.neon.tech/neondb?sslmode=require
   ```
3. **保存这个字符串**（稍后会用）

#### 4.3 初始化数据库

**方法 1：使用 Neon SQL Editor（推荐）**

1. 在 Neon 控制台点击 `SQL Editor`
2. 复制你的 `init_database.sql` 内容：
   ```bash
   cat init_database.sql
   ```
3. 粘贴到 SQL Editor
4. 点击 `Run`
5. 等待执行完成

**方法 2：使用命令行**

```bash
# 安装 PostgreSQL 客户端
sudo apt-get install postgresql-client

# 连接数据库（替换为你的连接字符串）
psql "postgresql://username:password@ep-xxx.region.aws.neon.tech/neondb?sslmode=require"

# 执行初始化脚本
\i init_database.sql

# 验证
\dt

# 退出
\q
```

---

### 步骤 5：部署到 Vercel ⏱️ 5分钟

#### 5.1 注册 Vercel

1. **访问**：https://vercel.com
2. **点击**：`Sign Up`
3. **选择**：`Continue with GitHub`
4. **授权** Vercel

#### 5.2 导入项目

1. **点击**：`Add New...` → `Project`
2. **找到**：`wogan-project` 仓库
3. **点击**：`Import`

#### 5.3 配置项目

1. **Project Name**: `wogan-project`
2. **Framework Preset**: `Other`
3. **Build Settings**:
   - Build Command: `npm run build`
   - Output Directory: `build/client`
   - Install Command: `npm install`

#### 5.4 添加环境变量

**在部署前，点击 `Environment Variables`**：

添加变量 1：
- Name: `DATABASE_URL`
- Value: （粘贴你的 Neon 连接字符串）
- 选择：`Production`, `Preview`, `Development`（全选）

添加变量 2：
- Name: `NODE_ENV`
- Value: `production`
- 选择：`Production`

#### 5.5 开始部署

1. **点击**：`Deploy`
2. **等待**：2-5 分钟
3. **查看**：部署日志

## ❓ 常见问题解决

### Q1：npm 安装失败

```bash
# 清除缓存
npm cache clean --force

# 重新配置镜像
npm config set registry https://registry.npmmirror.com

# 重试
npm install
```

### Q2：Git 推送失败

```bash
# 检查远程仓库
git remote -v

# 如果没有，重新添加
git remote add origin https://github.com/YOUR_USERNAME/wogan-project.git

# 强制推送
git push -u origin main --force
```

### Q3：Vercel 部署失败

**检查**：
1. 环境变量是否正确配置
2. 构建日志中的错误信息
3. 数据库连接字符串是否正确

**解决**：
1. 在 Vercel 控制台查看 `Deployments` → 点击失败的部署 → 查看日志
2. 修复错误后，推送代码会自动重新部署

### Q4：页面空白或 500 错误

**检查**：
1. 浏览器控制台（F12）是否有错误
2. Vercel Function Logs 是否有错误
3. 数据库是否初始化成功

**解决**：
```bash
# 重新初始化数据库
psql "your-neon-connection-string"
\i init_database.sql
```

### Q5：数据库连接失败

**检查**：
1. DATABASE_URL 环境变量是否正确
2. Neon 数据库是否正常运行
3. 连接字符串是否包含 `?sslmode=require`

**解决**：
1. 在 Vercel 重新设置 DATABASE_URL
2. 确保连接字符串格式正确
3. 重新部署
